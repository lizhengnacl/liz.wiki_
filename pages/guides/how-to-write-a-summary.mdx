# 如何写好工作总结

<Card title="【Internet+Talk】">
  本文转自美团内部自制的精品视频课程栏目【Internet+Talk】109期
</Card>

工作总结要围绕着读者关注的问题来展开，一般来说，读者关注的点包括业务思考、重点工作进展、工作中的问题和挑战以及下个阶段的工作计划。同时，工作总结要包含你的个人总结和FAQ。

## 一、业务思考

为了让业务繁忙的读者能理解我们接下来要阐述的工作，避免受到不必要的挑战，我们需要在介绍主要工作内容前，先说明我们对业务的思考。业务思考包括两个方面：

- <Color type='orange'>岗位的定位和价值</Color>：所在业务及个人岗位的定位和对公司产生的价值认知。
- <Color type='orange'>向外看（可选）</Color>：提供值得参考的行业信息，帮助读者设定一个标杆，清楚我们处于什么阶段，什么位置。

## 二、工作回顾

读者会希望了解关键的过程和策略，以及你在认知上的迭代。我们可以从四个方面进行工作回顾：

- <Color type='orange'>重点工作</Color>：结论先行，一句话总结重点工作是什么 。
- <Color type='orange'>关键策略</Color>：为完成工作采取了哪些关键策略和战术动作 。
- <Color type='orange'>工作进展</Color>：量化的结果，数据展示 。
- <Color type='orange'>认知迭代</Color>：对当前业务相对之前有哪些新的思考。

为了便于读者理解，我们可以用SCQA的结构来描述重点工作。

## 三、问题与挑战

对于任何一个团队来说，问题都是无穷多的。我们可以以终为始，从取得更好的成绩来讨论，讨论影响取得更好成绩的障碍。这些障碍可以从三个维度来考虑：

- <Color type='orange'>非常大的障碍</Color>：如，业务漏洞、技术漏洞。
- <Color type='orange'>外部障碍</Color>：如，跨部门协作中的问题、外部供应商问题。
- <Color type='orange'>需要别人解决的障碍</Color>：如，需要资源支持，需要上级或同事的协助。

## 四、下个阶段计划

很多同学在工作总结中会有下一步的工作目标，却没有实现路径，目标当然也就很难实现。我们在写下一步计划时，需要包括两个部分：

- <Color type='orange'>业务目标</Color>：接下来的业务目标是什么，要求是定性的、有挑战的，且不多于5条。
- <Color type='orange'>关键路径</Color>：你打算如何做，采取哪些手段和方法来实现业务目标。
